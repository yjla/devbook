文档流与元素类型

要理解两者区别，先得知道它们排在一个共同的舞台上——标准文档流。

标准文档流​

标准文档流（normal flow）是浏览器默认的排版规则：不写任何定位、浮动，元素就按它在 HTML 里的书写顺序，自上而下、自左到右地依次摆放。绝大多数页面的布局，本质都是在这个默认流的基础上做微调。

流里的元素分两类，排布方式完全不同：

  • 块级元素：独占一行，垂直方向从上往下堆叠
  • 行内元素：不换行，水平方向从左往右排列，一行排满才换行

float、position: absolute 这些做的事，就是把元素从标准流里拿出来，让它脱离默认排布——所谓「脱离文档流」。不脱离时，一切都遵循上面这套默认规则。

块级元素 block​

独占一行，默认宽度撑满父容器，可以自由设置宽高和四个方向的 margin/padding。

<div>我独占一行</div>
<p>我也独占一行，会另起一行</p>

常见块级元素：<div>、<p>、<h1>~<h6>、<ul>/<ol>/<li>、<section>/<article> 等语义标签、<form>。

行内元素 inline​

不独占一行，多个行内元素水平排在同一行，宽高由内容自然撑开。

<span>我</span><a>和它们</a><strong>排在同一行</strong>

行内元素有两个关键限制：

  1. 设置 width / height 无效——尺寸完全由内容决定
  2. 垂直方向的 margin/padding 不影响布局——padding-top、margin-bottom 视觉上可能撑大背景，但不会把上下的元素挤开；水平方向的 margin-left、padding-right 则正常生效

常见行内元素：<span>、<a>、<strong>/<em>、<b>/<i>、<label>、<code>。

行内块 inline-block​

兼具两者特点：像行内元素一样不独占行（水平排列），又像块级元素一样能设宽高、四向 margin/padding 都生效。

<button>按钮</button>
<input />
<img src="a.png" />

<img>、<input>、<button> 天生就是行内块的行为。普通元素也能用 display: inline-block 手动变成这种模式——早年做水平排列的导航栏常用它。

信息

这三种行为由 CSS 的 display 属性决定，可以互相转换：display: block / inline / inline-block。所谓「<div> 是块级、<span> 是行内」，只是浏览器默认样式表给它们设了对应的 display 值，并非标签写死的特性。

提示

<img> 是个特例：它在文档流里像行内元素一样参与排列（不独占行），却能设置宽高——因为它属于「替换元素」（replaced element），内容由外部图片资源决定，浏览器需要让你控制它的显示尺寸。<input>、<video> 同理。

行内元素的空白间隙​

把多个 inline-block 元素排在一起，它们之间会莫名多出几像素缝隙：

<span class="box"></span>
<span class="box"></span>

原因是 HTML 源码里标签之间的换行和空格被当成了一个空白字符渲染出来——行内元素会保留这种空白。常见解法：把标签写在同一行不留空格、给父元素设 font-size: 0、或干脆改用 flex 布局（flex 子项不受这种空白影响）。
