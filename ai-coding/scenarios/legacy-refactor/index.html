遗留系统重构

遗留系统重构是 AI 辅助开发中最有价值、也最复杂的场景。大量老旧代码缺乏文档、测试稀少、原作者已离职——这正是 AI 能发挥优势的地方，也是对 AI 能力的真实考验。

重构的五个阶段​

理解现有业务 → 模块拆分设计 → 建立测试防护网 → 代码迁移重构 → 验证与测试

这个顺序至关重要。跳过「建立测试防护网」直接重构，是遗留系统改造失败的最常见原因。

第一阶段：理解现有业务​

AI 在这里最直接的价值是加速代码阅读：

你：这个 OrderService 是做什么的？它的主要职责有哪些？
AI：分析整个文件后给出摘要，列出主要方法和依赖

用到的工具：

  • Sourcery / Semgrep：静态分析，识别代码模式和反模式
  • Agentic Chat：问 AI「这个模块的数据流是什么」，让它自己找相关文件
  • sqlc / Jooq：将数据库 schema 反向生成代码类型，理解数据结构

典型提问方式​

1. 「给我生成这个模块的调用链图」
2. 「找出所有直接操作数据库的地方」
3. 「这个类有哪些副作用？」
4. 「哪些地方耦合最严重？」

第二阶段：模块拆分设计​

理解完整系统后，和 AI 一起设计新的模块边界：

现有：一个 5000 行的 UserController
目标：拆分为 UserService + AuthService + ProfileService

提示 AI：
根据以下代码，建议如何按单一职责原则拆分模块，
并列出每个模块的对外接口

注意：AI 给出的拆分方案需要人工审查，它不了解你的业务演进方向。

第三阶段：建立测试防护网​

重构前必须先有测试，否则无法验证重构前后行为一致。

AI 辅助测试生成​

你：为 processOrder 方法生成特征测试（Characterization Test）
AI：分析方法行为，生成覆盖当前实际行为的测试用例

特征测试（Characterization Test） 的特殊之处：不是测「应该怎样」，而是测「现在实际怎样」——先锁定当前行为，再安全重构。

Testcontainers​

对于依赖数据库的遗留代码，用真实数据库跑测试：

const container = await new PostgreSqlContainer().start()
// 用真实 PostgreSQL 跑测试，而不是 mock

第四阶段：代码迁移重构​

有了测试保护，才能开始实际重构。

增量迁移，而非大爆炸​

❌ 一次性重写整个模块
✅ 每次只迁移一个函数，跑测试，再迁移下一个

AI 在这里的角色：

  • 将旧写法转换为新风格（如 callback → async/await）
  • 按照新的模块边界拆分代码
  • 生成迁移后的类型定义

LST + LLM（针对大型系统）​

对于企业级重构，Lossless Semantic Trees（LST） 是关键技术：

源代码 → LST（保留所有语义信息的 AST）→ LLM 转换 → 输出代码

普通 AST 工具处理代码时会丢失注释、格式等信息，LST 保留所有细节，确保转换后代码仍然可编译。

Moderne 的 Moddy 就是用这套方案处理数十亿行代码的多仓库迁移。

COBOL → Java 的案例​

IBM Watsonx Code Assistant for Z 展示了跨语言迁移的流程：

COBOL 代码 → AI 解释（中文说明业务逻辑）→ 人工确认 → AI 生成 Java → 测试验证

关键是「AI 解释」这一步——让人工先理解业务逻辑，再决定怎么用 Java 重新实现。

第五阶段：验证​

行为一致性验证​

旧系统接口 + 新系统接口 → 对同一批输入运行 → 对比输出差异

Applitools 等视觉测试工具可以对比 UI 级别的变化。

性能基准​

重构后性能不应该退化：

重构前：跑基准测试，记录 p50/p95/p99 延迟
重构后：对比数据，超过阈值的需要排查

工具汇总​

阶段工具用途
理解代码Sourcegraph Cody代码问答、依赖分析
理解代码Semgrep模式识别、反模式检测
测试生成GitHub Copilot / Claude特征测试生成
测试环境Testcontainers真实数据库集成测试
代码转换OpenRewrite确定性规则转换
大规模迁移Moderne (LST)多仓库代码迁移
视觉验证ApplitoolsUI 回归对比

核心原则​

理解先于重构——不理解代码就开始重构，AI 帮你更快地制造混乱。

测试是安全网——没有测试的重构是在走钢丝，AI 帮你生成测试，但你要检查它们是否真的覆盖了关键路径。

增量迭代——每次小改动 + 跑测试，比一次大重构更安全，也更容易定位问题。
