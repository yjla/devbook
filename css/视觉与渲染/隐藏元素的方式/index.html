隐藏元素的方式

隐藏一个元素，最常用的是 display: none、visibility: hidden、opacity: 0 三种。它们看起来效果相似，但在是否占据空间、能否点击、能否被屏幕阅读器读取、触发重排还是重绘、子元素能否反向显示这几个维度上差别很大，选错会导致布局错乱或无障碍问题。

三种主要方式对比​

维度display: nonevisibility: hiddenopacity: 0
是否占据空间否，脱离文档流是，占位但不可见是，占位但不可见
响应点击事件否否是，仍能点击
屏幕阅读器读取否，无障碍树移除否是，仍会读出
触发重排/重绘重排 + 重绘仅重绘仅重绘（透明度走合成层，常只触发合成）
子元素能否反向显示否能，子元素设 visibility: visible 即可否，opacity 子元素无法超过父级
能否参与过渡动画否，display 不可过渡是，但 visibility 是离散切换是，可平滑过渡
信息

visibility 的「子元素可反向显示」是它独有的特性：父元素 visibility: hidden 后，单独把某个子元素设 visibility: visible，这个子元素会重新显示。display: none 和 opacity: 0 都做不到——前者整棵子树都不渲染，后者透明度是逐层叠加的（父级 0.5、子级 1，实际仍是 0.5）。

为什么 opacity: 0 仍能点击​

opacity 只改变像素的透明度，元素的几何盒子和命中区域（hit-testing region）完全保留。所以一个 opacity: 0 的按钮仍然能被鼠标点到、被键盘 Tab 聚焦、被屏幕阅读器读出。这既是坑（误触不可见元素）也是用法（自定义文件上传：把原生 <input type="file"> 设为 opacity: 0 盖在好看的按钮上，点击穿透到真实 input）。

如果想要「看不见且点不到」，配合 pointer-events: none：

.fade-out {
  opacity: 0;
  pointer-events: none;
}

其他隐藏方式​

方式写法特点
移出屏幕position: absolute; left: -9999px视觉消失但仍在文档流（绝对定位脱流）、可聚焦、屏幕阅读器可读
裁剪clip-path: inset(50%) 或 clip-path: circle(0)裁掉所有像素，占位但不可见
尺寸归零width: 0; height: 0; overflow: hidden内容被裁剪，常配合 padding: 0; border: 0
hidden 属性<div hidden>HTML 全局属性，等价于 display: none，但语义更清晰且可被 JS 直接读写

无障碍专用的「视觉隐藏」​

一个常见需求：内容对视觉用户隐藏，但要保留给屏幕阅读器（比如图标按钮的文字说明）。display: none 和 visibility: hidden 会一并从无障碍树移除，不能用。标准做法是 .sr-only 类：

.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}
<button>
  <svg aria-hidden="true"><!-- 图标 --></svg>
  <span class="sr-only">关闭弹窗</span>
</button>

适用场景​

  • 彻底移除、不占位：用 display: none（或 hidden 属性）。条件渲染、Tab 切换面板首选。
  • 占位但隐藏：用 visibility: hidden。比如保持布局稳定、避免内容隐藏后页面跳动。
  • 显隐动画/淡入淡出：用 opacity（配合 transition）。
注意

做淡入淡出动画时不能用 display: none，因为 display 是离散属性、无法被 transition 过渡，从 none 到 block 会瞬间切换、没有动画。

正确的「淡出后移除」要把 opacity（可过渡）和 visibility（控制不可点击）一起用，等过渡结束再切 display：

.modal {
  opacity: 1;
  visibility: visible;
  transition:
    opacity 0.3s,
    visibility 0s 0s;
}
.modal.hidden {
  opacity: 0;
  visibility: hidden;
  /* visibility 延迟到 opacity 过渡结束后才切换，保证淡出全程可见 */
  transition:
    opacity 0.3s,
    visibility 0s 0.3s;
}
提示

现代浏览器支持给 display 加过渡：配合 transition-behavior: allow-discrete 和 @starting-style，可以让 display: none 也参与淡入淡出。但兼容性还在爬坡，生产环境仍以 opacity + visibility 方案最稳。
