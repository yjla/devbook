常见效果实现

汇总几个高频的「手写 CSS」效果，每个都给出可以直接用的代码。

1px 边框​

问题：在 devicePixelRatio > 1 的高清屏上，CSS 的 1px 会被渲染成多个物理像素，看起来比设计稿粗。

设计稿里的 1px 通常指物理像素，而 CSS 的 1px 是逻辑像素。在 2 倍屏上 1 个逻辑像素等于 2 个物理像素，所以边框看着偏粗。

成因

devicePixelRatio = 物理像素 / 逻辑像素。2 倍屏 dpr = 2，3 倍屏 dpr = 3。CSS 的尺寸单位是逻辑像素，浏览器把 1px 映射成 dpr 个物理像素来绘制，于是边框变粗。

方案一：伪元素 + transform: scale（最通用）

放一个铺满父元素的伪元素，给它画正常边框，再整体缩放 0.5，边框就变成物理上的半像素，视觉上接近 1 物理像素。

.border-1px {
  position: relative;
}

.border-1px::after {
  content: '';
  position: absolute;
  left: 0;
  top: 0;
  width: 200%;
  height: 200%;
  border: 1px solid #ddd;
  border-radius: 8px;       /* 圆角同样要 ×2 */
  transform: scale(0.5);
  transform-origin: 0 0;    /* 以左上角为基准缩放，避免位移 */
  box-sizing: border-box;
  pointer-events: none;
}

用 width/height: 200% + scale(0.5) 而不是直接 scale(0.5) 一个同尺寸盒子，是为了让缩放后伪元素仍然铺满父元素。

只需要单边边框时，把 border 换成对应的单边即可：

.border-bottom-1px::after {
  /* 仅底边 */
  width: 100%;
  height: 1px;
  border: none;
  background: #ddd;
  transform: scaleY(0.5);
  transform-origin: 0 100%;
}

方案二：box-shadow

用 inset 阴影模拟边框，阴影可以做到亚像素：

.border-shadow {
  box-shadow: inset 0 0 0 0.5px #ddd;
}

方案三：border-image

切一张 1 物理像素的边框图，用 border-image 平铺。兼容性最好但维护成本高，现在用得少。

.border-image {
  border-width: 1px;
  border-image: url(border.png) 2 stretch;
}
建议

优先用方案一（伪元素 + scale），可控性强且不依赖图片。只画底/顶分割线时，单边方案最简洁。

文字省略号​

单行省略：三件套缺一不可。

.ellipsis {
  overflow: hidden;
  white-space: nowrap;
  text-overflow: ellipsis;
}
  • white-space: nowrap 让文字不换行
  • overflow: hidden 裁掉超出部分
  • text-overflow: ellipsis 把裁掉的位置显示成 ...

多行省略：用 -webkit-line-clamp，需要配合 -webkit-box。

.ellipsis-multi {
  display: -webkit-box;
  -webkit-box-orient: vertical;
  -webkit-line-clamp: 3;     /* 显示 3 行，超出部分省略 */
  overflow: hidden;
}
注意

多行省略依赖 -webkit- 私有属性，但主流浏览器（含 Firefox）都已支持，移动端可放心用。注意必须同时写 display: -webkit-box、-webkit-box-orient: vertical 和 overflow: hidden，少一个都不生效。

画三角形​

原理：盒子的四条边框在内容尺寸为 0 时，会在中心交汇成四个三角形。把不需要的边设为 transparent，只留一条边，就得到一个三角形。

.triangle {
  width: 0;
  height: 0;
  border: 50px solid transparent;
  border-bottom-color: #3498db;   /* 只保留下边 -> 朝上的三角形 */
}

上面这个是等腰三角形：底宽 100px（左右各 50px），高 50px。改变各边宽度可以调整形状。

直角三角形：把相邻的一条边设为 transparent，只留下两条相邻边中的一条。

.triangle-right {
  width: 0;
  height: 0;
  border-top: 50px solid #3498db;
  border-right: 50px solid transparent;
  /* 左下直角，斜边朝右上 */
}

各方向三角形的对照：

朝向保留的边
朝上 ▲border-bottom-color
朝下 ▼border-top-color
朝左 ◀border-right-color
朝右 ▶border-left-color

画圆 / 半圆 / 扇形​

核心是 border-radius。

圆形：正方形 + border-radius: 50%。

.circle {
  width: 100px;
  height: 100px;
  border-radius: 50%;
  background: #3498db;
}

半圆：宽是高的 2 倍，只让上半部分两个角变圆。

.semicircle {
  width: 100px;
  height: 50px;
  border-radius: 50px 50px 0 0;   /* 左上 右上 右下 左下 */
  background: #3498db;
}

扇形（1/4 圆）：正方形，只让一个角变圆。

.sector {
  width: 100px;
  height: 100px;
  border-radius: 100px 0 0 0;     /* 只圆左上角 -> 朝右下的扇形 */
  background: #3498db;
}
border-radius 的四值写法

border-radius: 左上 右上 右下 左下，顺时针。值是圆角半径，等于对应边长时该角变成完整的 1/4 圆弧。

理解 line-height​

line-height 是行高，即一行文字所占的垂直高度。它决定了相邻两行基线之间的距离。

与单行垂直居中的关系：当元素只有一行文字时，让 line-height 等于 height，文字就会垂直居中。

.btn {
  height: 40px;
  line-height: 40px;   /* 单行文字垂直居中 */
}

原理：行高产生的空白会平均分配到文字上下（上下各半，称为半行距 half-leading），所以 line-height === height 时文字正好居中。

仅限单行

这招只对单行有效。多行时每一行都是 40px 行高，整体会超出 height，应改用 Flexbox 居中。

1.5 vs 150% / 1.5em 的继承差异：这是高频考点。

写法继承的值子元素表现
line-height: 1.5（无单位）继承倍数 1.5子元素按自己的 font-size × 1.5 计算
line-height: 150%继承计算后的具体值用父元素 font-size 算出的固定值传给子元素
line-height: 1.5em继承计算后的具体值同上，固定值

举例：父元素 font-size: 20px，子元素 font-size: 40px。

  • 父元素写 line-height: 1.5：子元素行高 = 40 × 1.5 = 60px，正常。
  • 父元素写 line-height: 150%：父元素先算出 20 × 1.5 = 30px，把 30px 这个固定值继承给子元素。子元素 font-size: 40px 但行高只有 30px，文字会重叠。
/* 推荐：无单位，让每个子元素按自己字号计算行高 */
body {
  line-height: 1.5;
}
建议

全局行高用无单位的 line-height: 1.5，避免大字号子元素行高被父元素算死导致文字重叠。
