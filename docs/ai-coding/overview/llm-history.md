# 大语言模型简史

LLM (大语言模型，Large Language Model) 是用超大规模神经网络让机器理解、生成人类语言的深度学习模型，发展至今大致经历三个阶段。

## 2017–2021：学术奠基

这一阶段的成果主要停留在论文和研究机构内，普通用户几乎感知不到。

**2017年6月**，Google 发布论文[《Attention Is All You Need》](https://arxiv.org/abs/1706.03762)，提出 **Transformer** 架构，注意力机制解决了并行计算和长距离依赖问题，奠定了后续所有大模型的基础。

**2018–2020年**，OpenAI 的 **GPT** 系列与 Google 的 **BERT** 等模型相继出现，确立了「**大规模预训练 + 任务微调**」范式——先在海量语料上训练通用能力，再针对具体任务微调，通用语言能力得以迁移复用。

**2021年7月**，OpenAI 发布 **Codex**，专门针对代码训练，第一次证明 LLM 能写出可运行的代码，HumanEval 通过率 28.8%。

:::info[Codex 命名混淆]
2025 年 OpenAI 复用了「Codex」这个名字，发布了一个 Agent 编程工具，与此处的 2021 模型是完全不同的东西。
:::

## 2022–2024：走向大众

AI 从实验室走向大众，任何人都能使用。

**2022年6月**，GitHub 联合 OpenAI 推出 **GitHub Copilot**，基于 Codex 模型，是第一个面向开发者的商业化 AI 编程工具。

**2022年11月**，OpenAI 发布 **ChatGPT**，两个月一亿用户。它第一次让普通人不需要懂技术就能与 AI 对话——AI 从专业工具变成了大众入口。

**2023年3月**，OpenAI 发布 **GPT-4**，第一个让人觉得输出「可以认真对待」的模型，推理能力大幅提升，幻觉明显减少。同月，百度发布**文心一言**，国内大模型热潮由此开启，阿里**通义千问**、字节**豆包**、腾讯**元宝**等相继跟进。

**2024年3月**，Anthropic 发布 **Claude 3**，首次引入 Opus / Sonnet / Haiku 分级产品策略，上下文扩展至 200K token，AI 开始能处理完整代码库。同年 6 月，**Claude 3.5 Sonnet** 在编码基准 SWE-bench Verified 上达到 **49%**，是编码能力的质变节点。

**2024年12月**，深度求索发布 **DeepSeek V3**，性能接近 GPT-4o，完全开源，国内模型首次引发全球广泛关注。

## 2025 至今：Agent 时代

从「AI 给建议，人来执行」变成「AI 自主完成任务」。

**2025年2月**，Anthropic 发布 **Claude Code**，在终端里给 AI 一个任务，它自己读代码、写代码、跑测试、提 PR，标志着 Agent 时代的开启。SWE-bench Verified 得分 **72%**，较一年前提升近 50 个百分点。

**2025年5月**，OpenAI 发布 **Codex** (Agent 版)，在云端沙箱中自主执行编程任务，与 Claude Code 定位相同，标志着各大厂商全面进入 Agent 编程赛道。

**2026年1月**，**OpenClaw** 开源走红，突破 10 万 GitHub stars。本地运行，将 LLM 与用户电脑直接打通——读写文件、执行终端命令、控制浏览器、调用 API，用自然语言驱动整套工作流。

**2026年2月**，Nous Research 发布 **Hermes Agent**，10 周内突破 10 万 GitHub stars。核心特点是自我进化——Agent 根据任务结果持续创造和优化自己的技能，同时具备跨会话的长期记忆（语义记忆 + 工作记忆 + 情节日志）。模型无关，支持 OpenAI、Ollama 等多种后端。

竞争焦点已从「模型够不够聪明」转向「Agent 能不能真正落地」。

:::info[什么是 SWE-bench]
SWE-bench 是一个 AI 编程能力基准：给模型一个真实 GitHub 仓库 + Issue，让它生成修复代码 (patch)，以能否让失败的测试用例通过作为唯一验收标准。得分代表解决率，2024 年前接近 0%。
:::
