# 中间件



中间件的设计模式：

```js
function logger(store) { // 接收store作为参数，为了解构出dispatch, getState等函数
  return function(next) { // next指的是老的dispatch函数
    return function(action) { // 这里返回的是新的dispatch函数
      console.group(action.type);
      console.info('dispatching', action);
      let result = next(action);
      console.log('next state', store.getState());
      console.groupEnd();
      return result
    }
  }
}
```





## Redux-Thunk

在使用 Redux-Thunk 前，传给 `dispatch` 的 `action` 必须是一个纯对象。使用 Redux-Thunk 后，传给 `dispatch` 的 `action` 可以是一个函数，这个函数会传入 `dispatch` 本身作为参数。



例子：

```js
import { createStore, applyMiddleware } from 'redux'
import thunk from 'redux-thunk'

const store = createStore(
  reducer,
  applyMiddleware(thunk)
)

store.dispatch({ type: 'INCREMENT' })

store.dispatch(function (dispatch) {
  dispatch({ type: 'INCREMENT' })
  dispatch({ type: 'INCREMENT' })
  dispatch({ type: 'INCREMENT' })

  setTimeout(() => {
    dispatch({ type: 'DECREMENT' })
  }, 1000)
})
```



Redux-Thunk 源码：

```typescript
function createThunkMiddleware<
  State = any,
  BasicAction extends Action = AnyAction,
  ExtraThunkArg = undefined
>(extraArgument?: ExtraThunkArg) {
  const middleware: ThunkMiddleware<State, BasicAction, ExtraThunkArg> =
    ({ dispatch, getState }) =>
    next =>
    action => {
      if (typeof action === 'function') {
        return action(dispatch, getState, extraArgument)
      }
      return next(action)
    }
  return middleware
}
```
