# 继承

继承要同时做对两件事：**属性各自独立** (子类每个实例的属性互不影响)，**方法共享复用** (父类原型上的方法只存一份)。JS 继承方案的演进史，就是一步步把这两点都凑齐的过程。

形象的例子：把继承想成 **开连锁加盟店**。父类是总部，子类是加盟店。属性独立，是指每家店的「现金、库存」各算各的，A 店进货不能影响 B 店；方法共享，是指总部的「运营手册」(原型方法) 所有店共用一本，不必每开一家店就重印一本。

## 几种方案的演进

### 原型链继承：方法能共享，但属性被所有实例共用

```js
function Parent() {
  this.hobbies = ['读书'];
}
Parent.prototype.say = function () {
  console.log('hi');
};

function Child() {}
Child.prototype = new Parent(); // 子类原型 = 一个父类实例
```

问题：所有子类实例**共用同一个父类实例**，引用类型的属性 (`hobbies`) 一改全改。相当于所有加盟店共用一个库存账本。

```js
const a = new Child();
const b = new Child();
a.hobbies.push('打球');
console.log(b.hobbies); // ['读书', '打球'] —— b 也被连累了
```

### 借用构造函数：属性独立了，但方法继承不到

```js
function Child() {
  Parent.call(this); // 在子类里执行父类构造函数，属性就变成自己的了
}
```

属性问题解决了，但只 `call` 了构造函数、没接上原型，父类**原型上的方法继承不到**。相当于每家店自己记账 (属性独立) 了，却没拿到总部的运营手册。

### 组合继承：两者结合，但有小浪费

```js
function Child() {
  Parent.call(this); // 第二次调用 Parent：拿到独立属性
}
Child.prototype = new Parent(); // 第一次调用 Parent：接上原型方法
Child.prototype.constructor = Child;
```

能用，但 **`Parent` 被调用了两次**，子类原型上残留了一份永远用不到的父类属性，有浪费。

### 寄生组合继承：最优解

思路：把「接原型」那一步从 `new Parent()` 换成 `Object.create(Parent.prototype)`——只复制原型这本「运营手册」，不再把总部 (父类构造函数) 重新跑一遍，从而砍掉组合继承里多余的那次调用。

```js
function inherit(Child, Parent) {
  // 第一步：只拿父类的原型，不执行父类构造函数
  Child.prototype = Object.create(Parent.prototype);
  // 第二步：修正 constructor 指向，让它重新指回 Child
  Child.prototype.constructor = Child;
}

function Parent(name) {
  this.name = name;
  this.hobbies = ['读书'];
}
Parent.prototype.say = function () {
  console.log(this.name);
};

function Child(name, age) {
  Parent.call(this, name); // 属性独立
  this.age = age;
}

inherit(Child, Parent); // 方法继承，且 Parent 只在 new 实例时被调一次
Child.prototype.study = function () {
  console.log('study');
};
```

:::tip
寄生组合继承是 ES6 之前的「标准答案」。关键就一句：**用 `Object.create(Parent.prototype)` 接原型**，避免组合继承里多余的那次父类构造函数调用。
:::

## ES6 class extends (语法糖)

`class` + `extends` 底层就是寄生组合继承，但写法清爽得多。

```js
class Parent {
  constructor(name) {
    this.name = name;
    this.hobbies = ['读书'];
  }
  say() {
    console.log(this.name);
  }
}

class Child extends Parent {
  constructor(name, age) {
    super(name); // 相当于 Parent.call(this, name)，必须在用 this 之前调用
    this.age = age;
  }
  study() {
    console.log('study');
  }
}
```

:::warning
`super(name)` 必须在子类构造函数里**先于 `this` 调用**——因为子类的 `this` 要靠父类构造函数初始化出来，不先 `super` 就用 `this` 会直接报错。这是 `class` 和寄生组合继承在语义上的一个重要区别。
:::
