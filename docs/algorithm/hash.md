# 哈希

**结论先行**：哈希的核心价值就一句话——**用空间换时间，把「查找某个东西在不在」从 O(n) 降到 O(1)**。

普通数组想知道「5 在不在里面」，得从头扫一遍，O(n)。哈希表靠一个哈希函数把「键」直接换算成存储位置，不用扫，平均一步到位。增、删、查平均都是 O(1)，代价是额外开一块内存。

:::info
**生活类比：图书馆找书 vs 储物柜取包**。在没有索引的书堆里找一本书，得一本本翻（O(n)）。但车站储物柜不一样——你存包时拿到 37 号牌，取包时直接走到 37 号柜，不用挨个柜子开。「凭钥匙一步到位」就是哈希：键（牌号）经过换算直接定位到存储格子。
:::

## 什么时候该想到哈希

看到下面这些信号，第一反应就该是哈希：

- 需要反复判断 **某元素是否出现过 / 出现几次** → 用哈希表计数。
- 需要给元素 **配对**，比如「找另一个数凑成目标」 → 用哈希表存「见过的数」，回头查它的搭档。
- 需要 **去重**，或者快速判断 **某个值在不在集合里** → 用 Set。
- 一句话：凡是发现自己在写「嵌套循环去找匹配」，多半能用哈希把内层那一圈干掉。

:::info
JavaScript 里三种哈希结构：

- **Map**：键值对映射，键可以是任意类型，最通用。
- **Set**：只存键、不存值的集合，专门用来去重和判断存在。
- **普通对象 `{}`**：也能当哈希表，但键会被转成字符串，且有原型链干扰，刷题优先用 `Map` / `Set` 更干净。
:::

## 例题一：两数之和 (1)

**题目**：给一个数组和目标值 `target`，找出数组里两个数，它们之和等于 `target`，返回这两个数的下标。

暴力解是双重循环，每个数去试后面所有数，O(n²)。哈希能把它降到 O(n)——关键转念：与其「找两个数」，不如固定当前这个数 `x`，去 **找它缺的那一半 `target - x`**。这个「找」就交给哈希表做 O(1)。

:::info
**生活类比：拼图配对**。你手里一块拼图，要在桌上找能咬合的那一块。聪明的做法不是把桌上每两块两两比，而是边翻边把翻过的拼图按形状记进脑子；翻到新一块时，先回想「我刚才有没有见过能配它的形状」。见过就立刻配上，没见过就把这块也记下。「先回想、再记下」对应哈希的「先查、再存」。
:::

**解题思路 1-2-3**：

```js
function twoSum(nums, target) {
  // 第一步：准备一个哈希表，存「已经见过的数 → 它的下标」
  const seen = new Map();

  // 第二步：从左到右遍历每个数 x
  for (let i = 0; i < nums.length; i++) {
    const x = nums[i];
    const need = target - x; // x 缺的那一半

    // 第三步：先查——表里有没有 need？有就说明搭档在前面出现过，直接返回
    if (seen.has(need)) {
      return [seen.get(need), i];
    }

    // 第四步：没查到，再把当前数和下标存进去，留给后面的数来配
    seen.set(x, i);
  }

  return [];
}
```

走一遍 `nums = [2, 7, 11, 15]`, `target = 9`：

| 当前数 x | 缺的另一半 (9 - x) | 哈希表里有吗 | 动作 |
|---|---|---|---|
| 2 | 7 | 没有 | 存入 `{2: 0}` |
| 7 | 2 | **有，在下标 0** | 返回 `[0, 1]` |

:::warning
顺序必须是 **先查再存**。如果先存再查，遇到 `target` 正好是 `x` 两倍时，会把自己当成搭档（同一个下标用了两次）。先查再存天然避开这个坑——查的时候哈希表里只有「我前面的数」。
:::

## 例题二：最长连续序列 (128)

**题目**：给一个 **没排序** 的数组，找出数值上连续的最长一段有多长。比如 `[100, 4, 200, 1, 3, 2]` 里 `1,2,3,4` 连成一串，答案是 4。注意「连续」指数值挨着，和位置无关，且要求 O(n)。

排序后扫一遍当然能做，但那是 O(n log n) 不达标。O(n) 的办法是用 Set，配一个关键洞察：**一段连续序列，只从它的「起点」开始往上数**。

怎么判断 `x` 是不是起点？看 `x - 1` 在不在 Set 里——不在，`x` 就是开头；在，说明 `x` 只是中间某个数，跳过它，留给真正的起点去数。

:::info
**生活类比：排查多米诺骨牌的最长一串**。地上散落着标了号的骨牌，你想找连号最长的一串。聪明做法是只从「每串的第一张」开始数：判断一张牌是不是某串的开头，就看「它前一号的牌存不存在」——前一号不在，它就是开头，从它往后一张张数到断为止。这样每张牌只会在它所在那串被数到一次，绝不重复劳动。
:::

**解题思路 1-2-3**：

```js
function longestConsecutive(nums) {
  // 第一步：所有数丢进 Set，顺手去重，并让「某数在不在」变成 O(1) 查询
  const set = new Set(nums);
  let longest = 0;

  // 第二步：遍历每个数 x
  for (const x of set) {
    // 第三步：只在 x-1 不在 Set 里时，才认定 x 是某段序列的起点
    if (set.has(x - 1)) {
      continue; // x 只是中间的数，跳过，留给真正的起点去数
    }

    // 第四步：从起点出发，不断查 x+1、x+2…… 直到断掉，记录这段长度
    let cur = x;
    let length = 1;
    while (set.has(cur + 1)) {
      cur++;
      length++;
    }

    // 第五步：全程取最大值
    longest = Math.max(longest, length);
  }

  return longest;
}
```

走一遍 `[100, 4, 200, 1, 3, 2]`，Set = `{100, 4, 200, 1, 3, 2}`：

| 当前数 x | x-1 在 Set？ | 是起点吗 | 动作 |
|---|---|---|---|
| 100 | 99 不在 | 是 | 往上数：101 不在，长度 1 |
| 4 | 3 在 | 不是 | 跳过 |
| 200 | 199 不在 | 是 | 往上数：201 不在，长度 1 |
| 1 | 0 不在 | 是 | 往上数 2→3→4→5不在，长度 **4** |
| 3 | 2 在 | 不是 | 跳过 |
| 2 | 1 在 | 不是 | 跳过 |

最长 = 4。注意 `1,2,3,4` 这条链 **只在遇到起点 1 时数了一次**，遇到 2、3、4 都直接跳过，没重复劳动——这正是它能做到 O(n) 的原因：每个数最多被「往上数」访问一次。

:::tip
很多人会担心「外层遍历 + 内层往上数」是不是 O(n²)。不是。因为内层只在起点才真正展开，整个数组所有的「往上数」加起来总共也只走 n 步，所以是 O(n)。「只从起点数」这条规则就是为了砍掉重复。
:::

## 小结

- 哈希的本质是 **空间换时间**，把查找从 O(n) 压到平均 O(1)。
- 触发信号：判断 **出现过没有 / 出现几次**、找 **配对**、**去重**——发现自己在写嵌套循环找匹配，就想哈希。
- **两数之和**：哈希表存「见过的数 → 下标」，边遍历边查「另一半」，记牢先查后存。
- **最长连续序列**：Set 去重 + O(1) 查存在，只从「`x-1` 不存在」的起点往上数，保证 O(n)。
- JS 里优先用 `Map`（键值映射）和 `Set`（去重 / 判断存在），比普通对象更干净。

> ## 一句话口诀
>
> 哈希就是空间换时间；两数之和「先查后存找另一半」；最长连续序列「只从没有左邻居的起点往上数」。
