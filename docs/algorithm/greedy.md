# 贪心算法

**贪心算法就是每一步都选当前看起来最划算的那个选项，选完不回头，指望这一连串的局部最优能拼出全局最优。** 它不像动态规划那样保存所有子问题的答案再权衡，而是「走一步看一步、只认眼前最优」。

:::tip 形象记忆
贪心像**逛超市凑满减、手里钱包找零**：要找 88 元，你会先抽最大的 50，再 20、再 10、再 5、再 2、再 1——每次都抓能用的最大面额，不会先抓小的再后悔。人民币面额经过精心设计，这么贪一定最省;但如果面额换成 `1, 5, 11`，找 15 元贪心会选 `11 + 1 + 1 + 1 + 1`(5 枚)，而最优是 `5 + 5 + 5`(3 枚)——**贪心就翻车了**。这正是贪心的命门:它对面额(问题结构)挑食。
:::

## 贪心成立的两个前提

一个问题能用贪心，必须满足：

- **贪心选择性质**：全局最优解可以通过一系列局部最优选择达到。即「当前这步选最优，不会堵死后面通向全局最优的路」。
- **最优子结构**：原问题的最优解包含子问题的最优解(这点和 DP 相同)。

:::warning
**贪心最大的坑是「看着对，其实错」**。它不保证正确，能不能用，关键看「贪心选择性质」成不成立——而这往往需要严格证明 (常用**交换论证法**:假设存在更优解，证明把它的某步换成贪心的选择不会变差)。面试里拿不准时，先想个反例试试;举不出反例、又能证明，才敢用贪心。
:::

## 贪心 vs 动态规划

两者都要求最优子结构，区别在**要不要回头权衡**：

| | 贪心 | [动态规划](./dynamic-programming.md) |
|---|---|---|
| 决策方式 | 每步选当前最优，**不回头** | 枚举所有子问题的组合，取最优 |
| 是否记表 | 不需要 | 需要 dp 表 |
| 正确性 | 需证明贪心选择性质 | 只要状态定义对就一定对 |
| 复杂度 | 通常更低 (常是排序 `O(nlogn)` 或线性) | 通常更高 (状态数 × 转移) |

:::info
能用贪心的题，DP 一般也能解，只是更慢;反过来不成立——DP 能解的题，贪心常常算错。所以**贪心是「能用就赚到」的优化**，DP 是「总归能用」的兜底。拿不准就先用 DP 保正确。
:::

## 经典题型

### 入门：分发饼干

有一群孩子和一堆饼干，孩子 `i` 的胃口是 `g[i]`，饼干 `j` 的大小是 `s[j]`。一块饼干只能给一个孩子，且大小 `≥` 胃口才能喂饱。求最多能喂饱几个孩子。

:::tip 形象记忆
像**用有限的礼物哄最多的小孩**：先让胃口最小的孩子上，拿刚好能满足他的最小饼干打发掉——把大饼干留给胃口大的。绝不用大饼干去喂小胃口的孩子，那是浪费。
:::

贪心策略：**胃口和饼干都从小到大排序，用最小的饼干去满足最小的胃口**。

```js
function findContentChildren(g, s) {
  // 第一步：两个数组都升序排，才能从小到大配对
  g.sort((a, b) => a - b);
  s.sort((a, b) => a - b);

  // 第二步：两个指针，child 指当前要喂的孩子，cookie 指当前饼干
  let child = 0;
  let cookie = 0;

  // 第三步：从小往大扫，喂得饱就两个指针都进，喂不饱就换更大的饼干
  while (child < g.length && cookie < s.length) {
    if (s[cookie] >= g[child]) {
      child++; // 这个孩子喂饱了
    }
    cookie++; // 无论喂没喂饱，这块饼干都用掉/跳过
  }

  // 第四步：child 走了多远，就喂饱了几个
  return child;
}
```

### 区间型：跳跃游戏

给一个非负整数数组 `nums`，`nums[i]` 表示从位置 `i` 最多能往前跳几步。问能否从第 0 位跳到最后一位。

:::tip 形象记忆
像**手机电量决定能走多远**：你不关心具体怎么走，只关心「**站在目前走过的每一格上，最远能摸到哪里**」。只要这个「最远触达点」一直能盖住脚下的路，就走得通;一旦脚下的格子超出了最远触达点，说明断电了，过不去。
:::

贪心策略：**实时维护「当前能到达的最远下标」，遍历时只要没被它落下，就不断刷新它**。

```js
function canJump(nums) {
  // 第一步：farthest 记录目前能跳到的最远下标，起点是 0
  let farthest = 0;

  // 第二步：逐格往前走
  for (let i = 0; i < nums.length; i++) {
    // 第三步：如果当前格已经超出能到达的范围，说明卡死了
    if (i > farthest) return false;
    // 第四步：用「站在 i 上能跳到的最远处」刷新全局最远点
    farthest = Math.max(farthest, i + nums[i]);
  }

  // 第五步：整个数组都没被甩下，说明能到终点
  return true;
}
```

### 区间调度：无重叠区间

给一堆区间 `intervals`，求**最少删掉几个**，能让剩下的区间互不重叠。

:::tip 形象记忆
像**一间会议室排满一天的会**：想塞进最多场会，就**每次都挑「最早散场」的那场**——结束得越早，给后面留的空越大。按结束时间贪心，能开的会最多，反过来要删的会就最少。
:::

贪心策略：**按区间结束位置升序排，依次保留与上一个不冲突的区间，冲突的就删掉**。

```js
function eraseOverlapIntervals(intervals) {
  if (intervals.length === 0) return 0;

  // 第一步：按结束位置升序排——结束越早，越该先保留
  intervals.sort((a, b) => a[1] - b[1]);

  // 第二步：保留第一个，记下它的结束位置；删除计数清零
  let prevEnd = intervals[0][1];
  let removed = 0;

  // 第三步：从第二个开始扫
  for (let i = 1; i < intervals.length; i++) {
    if (intervals[i][0] >= prevEnd) {
      // 第四步：起点不早于上一个的终点 → 不重叠，保留并更新终点
      prevEnd = intervals[i][1];
    } else {
      // 第五步：重叠了 → 删掉当前这个(它结束更晚，删它更划算)
      removed++;
    }
  }

  return removed;
}
```

## 小结

- 贪心是**每步只选当前最优、绝不回头**，能拼出全局最优的前提是**贪心选择性质 + 最优子结构**。
- 贪心的命门是**正确性**:看着对不一定真对，拿不准先举反例，能用前最好用交换论证法证明。
- 实现上贪心常落在**排序 + 一次扫描**:分发饼干两头排序配对、跳跃游戏维护最远触达点、区间调度按结束时间排序。
- 和 [DP](./dynamic-programming.md) 的取舍:贪心更快但挑问题，DP 更慢但总对——**拿不准就用 DP 兜底**。
