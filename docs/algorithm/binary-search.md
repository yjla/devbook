# 二分查找

**结论先行**：在 **有序** 数据上，每次比中间元素，一刀砍掉一半范围，把 `O(n)` 的线性查找压到 `O(log n)`。思路谁都懂，难在 **边界** 和 **循环终止条件**——稍有偏差不是死循环就是漏解。写对的唯一心法：**先想清区间怎么定义，再让每一行代码都和这个定义对齐**。

:::info
**生活类比：查字典**。要找「twilight」这个单词，你不会从第一页一页页翻，而是直接翻到字典中间，看到当前页是「m」开头——比 t 小，于是后半本继续，把前半本整本丢掉；再翻后半本的中间……每翻一次,要找的范围就少一半。这就是二分查找。前提是字典本来就是按字母排好序的——没排序就没法「翻中间」判断方向。
:::

## 结构：先定区间，再写循环

下面用最不易错的「左闭右闭」区间 `[left, right]`，即 `left` 和 `right` 指向的元素都算在搜索范围内。

```js
function binarySearch(nums, target) {
  // 第一步：定义区间 [left, right]，两端都是闭的，所以 right 指向最后一个元素
  let left = 0;
  let right = nums.length - 1;

  // 第二步：闭区间下 left == right 时区间里还有一个元素，所以条件要带等号
  while (left <= right) {
    // 第三步：取中点，用减法防止大数相加溢出（通用好习惯）
    const mid = left + Math.floor((right - left) / 2);

    // 第四步：拿中点和目标比，三种情况各自缩小区间
    if (nums[mid] === target) {
      return mid; // 正好命中
    } else if (nums[mid] < target) {
      left = mid + 1; // 目标在右半边，mid 已比过，跳过它
    } else {
      right = mid - 1; // 目标在左半边，同样跳过 mid
    }
  }

  // 第五步：区间走空还没找到，返回 -1
  return -1;
}
```

:::warning
三个最容易翻车的点，全部由「区间是闭的」这个定义推导出来，不要死记：

1. **`mid` 计算**：用 `left + (right - left) / 2`，不用 `(left + right) / 2`——后者大数会溢出（JS 数字虽不溢出，但这是通用习惯）。
2. **循环条件**：闭区间用 `while (left <= right)`，因为 `left == right` 时区间内还有一个元素要查。
3. **指针更新**：`mid` 已经比过了，所以要 `mid + 1` / `mid - 1` 把它跳过，否则区间不缩小，直接死循环。
:::

## 查找左右边界

当数组有重复元素，要找 **第一个** 或 **最后一个** 等于 target 的位置时，基础模板不够用。核心改动一句话：**命中后不立即返回，而是记下来，继续往一侧收缩**。

:::info
**生活类比：点名时找最早到的同名同学**。教室里坐了三个「张伟」，你要找坐得最靠前的那个。你不会一找到张伟就停手，而是记下他的座位号，然后继续往前排找——直到前面再没有张伟为止。最后记下的那个就是最靠前的。「找到也别停，往一侧继续找」正是边界二分的精髓。
:::

查找左边界（第一个等于 target 的下标）：

```js
function leftBound(nums, target) {
  // 第一步：照常定义闭区间，外加一个 res 记录「目前找到的最靠左位置」
  let left = 0;
  let right = nums.length - 1;
  let res = -1;

  while (left <= right) {
    const mid = left + Math.floor((right - left) / 2);

    // 第二步：命中后不返回，先记下来，再逼着区间往左继续找
    if (nums[mid] === target) {
      res = mid;
      right = mid - 1; // 关键：往左收缩，看左边还有没有更靠前的 target
    } else if (nums[mid] < target) {
      left = mid + 1;
    } else {
      right = mid - 1;
    }
  }

  return res;
}
```

查找右边界只需把命中后的收缩方向改成 `left = mid + 1`。掌握左右边界后，「在排序数组中查找元素的第一个和最后一个位置」这类题就是它的直接套用。

## 进阶：二分答案

二分的威力远不止「在数组里找数」。只要问题具备 **单调性**（随某个值变大，判定结果单调地从「不行」变成「行」），就能二分。

:::info
**生活类比：试水温调热水器**。你想要恰好能洗澡的最低水温。水温太低不行，太高了浪费电——而且有个规律：只要某个温度能洗，比它更高的一定也能洗（单调）。于是你不必一度一度试，而是直接试中间温度：能洗就往下调试更低的，不能洗就往上调，每次砍掉一半温度区间。这就是「对答案本身做二分」，常见于「最小的最大值」「在 D 天内送达的最低运力」这类题。
:::

- **搜索旋转排序数组**：数组被旋转过，但任意时刻 `mid` 的左半或右半必有一半是有序的，据此判断 target 在哪半。
- **二分答案**：答案落在一个有范围且单调的区间里，直接对答案值域二分，每次用一个 `check(x)` 函数判定 `x` 是否可行。

:::tip
判断一道题能不能二分，别只盯着「数组有没有序」，要问：**是否存在一个单调的判定关系**？只要「小了不行、大了就行」成立，哪怕题面里压根没有有序数组，也能对答案做二分。
:::

## 小结

- 二分前提是 **有序**（或更一般的 **单调判定关系**），复杂度 `O(log n)`。
- 写对的关键是 **区间定义自洽**：推荐「左闭右闭」+ `while (left <= right)` + `mid ± 1`。
- `mid` 用 `left + (right - left) / 2` 防溢出。
- 查找左右边界：命中后不返回，记下来继续往对应方向收缩。
- 进阶是 **二分答案**：只要判定关系单调，就能对答案值域二分，这是二分最有威力的用法。

> ## 一句话口诀
>
> 有序才能二分，闭区间配 `<=` 与 `mid ± 1`；找边界「命中不停、继续收缩」；判定单调就能对答案二分。
