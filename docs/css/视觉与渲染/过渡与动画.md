# 过渡与动画

`transition`（过渡）和 `animation`（动画）都能让样式变化平滑起来，区别在于：**`transition` 必须由状态变化触发、只有起止两个状态；`animation` 用 `@keyframes` 定义、能自动播放、支持多个关键帧、可循环。** 简单的 hover 变色用 `transition`，加载圈、跑马灯这类持续/复杂动画用 `animation`。

## transition vs animation

| 维度 | `transition` | `animation` |
| --- | --- | --- |
| 触发方式 | 需状态变化触发（hover、加 class、JS 改样式） | 可自动播放，无需触发 |
| 关键帧 | 只有起点和终点两态 | `@keyframes` 定义任意多个关键帧 |
| 循环 | 不能循环 | `animation-iteration-count` 可指定循环/无限 |
| 方向控制 | 无 | `animation-direction` 可反向、往返 |
| 结束后状态 | 回到触发前的样式（除非状态保持） | `animation-fill-mode` 控制保留首/尾帧 |
| 暂停 | 无法暂停 | `animation-play-state: paused` 可暂停 |
| 适用场景 | 简单交互：显隐、变色、缩放 | 复杂/持续：loading、轮播、强调动效 |

## transition

`transition` 是 `transition-property`、`transition-duration`、`transition-timing-function`、`transition-delay` 四个属性的简写：

| 子属性 | 作用 | 示例值 |
| --- | --- | --- |
| `transition-property` | 过渡哪个属性 | `transform`、`opacity`、`all` |
| `transition-duration` | 过渡时长 | `0.3s`、`300ms` |
| `transition-timing-function` | 速度曲线 | `ease`、`linear`、`ease-in-out`、`cubic-bezier(...)` |
| `transition-delay` | 延迟多久开始 | `0s`、`0.1s` |

简写顺序：`transition: 属性 时长 曲线 延迟`。

```css
.button {
  background: #3b82f6;
  transition: background 0.3s ease-in-out;
}
.button:hover {
  background: #2563eb;
}
```

多个属性分别设置时用逗号隔开：

```css
.card {
  transition:
    transform 0.3s ease,
    box-shadow 0.3s ease 0.1s;
}
```

:::warning
不要图省事用 `transition: all`。它会监听所有属性变化，可能意外过渡到一些你没想动的属性（比如某些会触发重排的属性），既影响性能也可能产生奇怪动效。明确写出要过渡的属性更可控。
:::

## animation 与 @keyframes

`@keyframes` 定义动画的关键帧序列，`animation` 把它应用到元素上：

```css
@keyframes spin {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}

.loading {
  animation: spin 1s linear infinite;
}
```

多关键帧用百分比：

```css
@keyframes bounce {
  0% {
    transform: translateY(0);
  }
  50% {
    transform: translateY(-20px);
  }
  100% {
    transform: translateY(0);
  }
}
```

`animation` 的主要子属性：

| 子属性 | 作用 | 示例值 |
| --- | --- | --- |
| `animation-name` | 关联的 `@keyframes` 名 | `spin` |
| `animation-duration` | 一次播放时长 | `1s` |
| `animation-timing-function` | 速度曲线 | `linear`、`ease` |
| `animation-delay` | 延迟开始 | `0.2s` |
| `animation-iteration-count` | 播放次数 | `1`、`3`、`infinite` |
| `animation-direction` | 播放方向 | `normal`、`reverse`、`alternate`、`alternate-reverse` |
| `animation-fill-mode` | 动画前后保留的样式 | `none`、`forwards`、`backwards`、`both` |
| `animation-play-state` | 运行/暂停 | `running`、`paused` |

:::info
`animation-fill-mode` 容易混淆：

- `forwards`：动画结束后**停在最后一帧**（最常用，否则元素会瞬间弹回初始样式）。
- `backwards`：在 `delay` 等待期间就**先应用第一帧**的样式。
- `both`：同时具备上面两者。
:::

`animation-direction: alternate` 让动画往返播放（奇数次正放、偶数次倒放），配合 `infinite` 做呼吸灯、来回浮动很常用：

```css
.breathing {
  animation: bounce 2s ease-in-out infinite alternate;
}
```

`animation-play-state` 常用来 hover 暂停：

```css
.marquee {
  animation: scroll 10s linear infinite;
}
.marquee:hover {
  animation-play-state: paused;
}
```

## 性能：优先动 transform 和 opacity

无论 `transition` 还是 `animation`，**优先选 `transform` 和 `opacity` 作为动画属性**。这两个属性可以由 GPU 在合成层处理，不触发重排、通常也不触发重绘，动画最流畅（60fps）。

反例是动 `width`、`height`、`top`、`left`、`margin` 这类几何属性，它们每一帧都触发重排，容易掉帧卡顿。

```css
/* 反例：动 left，每帧重排 */
@keyframes slide-bad {
  to {
    left: 300px;
  }
}

/* 正解：动 transform，走合成层 */
@keyframes slide-good {
  to {
    transform: translateX(300px);
  }
}
```

为什么会这样、重排重绘到底贵在哪里，见 [重排与重绘](./重排与重绘.md)。

:::tip
- 位移用 `transform: translate()` 代替 `top`/`left`/`margin`。
- 缩放用 `transform: scale()` 代替改 `width`/`height`。
- 淡入淡出用 `opacity` 代替 `visibility` 切换。
- 频繁动画的元素可加 `will-change: transform`（用完移除）提前提升合成层。
:::

## transform 常见函数

| 函数 | 作用 | 示例 |
| --- | --- | --- |
| `translate(x, y)` | 平移 | `translate(10px, 20px)`、`translateX(50%)` |
| `scale(n)` | 缩放 | `scale(1.2)`、`scaleY(0.5)` |
| `rotate(deg)` | 旋转 | `rotate(45deg)`、`rotate(-90deg)` |
| `skew(deg)` | 倾斜 | `skew(10deg, 5deg)` |

多个变换可串联，**注意顺序影响结果**（先平移再旋转 ≠ 先旋转再平移）：

```css
.box {
  transform: translateX(100px) rotate(45deg) scale(1.5);
}
```

:::info
`translate` 的百分比是相对**元素自身**的尺寸计算的，这点和 `margin`/`padding` 的百分比相对父元素不同。经典的水平垂直居中技巧 `top: 50%; left: 50%; transform: translate(-50%, -50%)` 正是利用了这一点——`-50%` 指元素自身宽高的一半。
:::
