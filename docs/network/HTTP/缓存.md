# 缓存

HTTP 缓存的是以前请求过的资源，比如 HTML 文档、图片、或者文件的响应。



## 缓存字段

按优先级排序：

1. `Cache-Control` 字段：通过其 `max-age` 属性，设置强缓存的过期时间，单位是秒。
2. `Expires` 字段：用于设置强缓存的过期时间，使用的是 UTC 时间，可能不准。
3. `If-None-Match/ETag` 字段：用于设置协商缓存的过期时间，精度高，但性能稍差（需要计算哈希值）。
4. `If-Modified-Since/Last-Modified` 字段：用于设置协商缓存的过期时间



## 缓存处理过程

1. 浏览器第一次请求某个资源
2. 如果服务器希望浏览器能缓存这次请求的资源，则会在返回的响应头中配置强缓存或协商缓存字段
3. 浏览器第二次请求该资源。在发起 HTTP 请求前，浏览器会先判断是否命中强缓存。
   - 如果命中强缓存，则返回200状态码，直接使用本地的缓存资源，不必再向服务器发起 HTTP 请求。
   - 如果不符合强缓存，则发起 HTTP 请求，带上 `If-None-Match` 或 `If-Modified-Since` 字段，服务器根据这两个字段验证是否符合协商缓存。
     - 如果命中协商缓存，服务器返回304状态码，让浏览器使用本地缓存。
     - 如果不命中协商缓存，服务器会返回修改后的资源。



## 强缓存和协商缓存的区别

协商缓存会发起HTTP请求，强缓存不会发起 HTTP 请求。



## 缓存优先级

1. Service Worker 缓存
2. 内存缓存
3. 磁盘缓存
4. 推送缓存



## 刷新对于缓存的影响

1. 在浏览器地址栏中直接输入 URL 跳转，浏览器会先检查强缓存，再检查协商缓存。
2. 按 F5 刷新网页时，浏览器会跳过强缓存，检查协商缓存。
3. 按 CTRL+F5 强制刷新网页时，浏览器会跳过强缓存和协商缓存，直接向服务器请求资源。
