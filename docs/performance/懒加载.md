# 懒加载

懒加载（lazy loading）是一种将资源标识为非阻塞资源并仅在需要时加载它们的策略。 这是一种缩短关键渲染路径长度的方法，可以缩短页面加载时间。比如说在一个图片很多，页面很长的电商网页中，我们可以让视口之外的图像不加载，只当这些图片滚动到视口内的时候才加载。

图片懒加载的实现方法：

- 设置 `<img>` 标签的属性 `loading="lazy"`
- 监听 `scroll` 事件，当根元素向下滚动的像素 `scrollTop` 加上视口高度`clientHeight` 大于图片的偏移量 `offsetTop` 时说明图片进入了视口
- 使用 `getBoundingClientRect()` 获取图片盒模型相对于视口的位置信息，如果小于视口高度 `clientHeight` 则说明该图片进入了视口
- 使用 Intersection Observer API 判断图片是否进入视口。Intersection Observer API 提供了一种异步检测目标元素与视口相交情况变化的方法。其他需要事件监听和频繁调用 `getBoundingClientRect()` 的方法会造成性能问题。



懒加载也叫延迟加载，指的是在长网页中延迟加载图片的时机，当用户需要访问时，再去加载，这样可以提高网站的首屏加载速度，提升用户的体验，并且可以减少服务器的压力。它适用于图片很多，页面很长的电商网站的场景。懒加载的实现原理是，将页面上的图片的 src 属性设置为空字符串，将图片的真实路径保存在一个自定义属性中，当页面滚动的时候，进行判断，如果图片进入页面可视区域内，则从自定义属性中取出真实路径赋值给图片的 src 属性，以此来实现图片的延迟加载。
