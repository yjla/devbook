# 作用域

作用域（scope）又被称为词法作用域，静态作用域，指的是变量存在的范围，在代码编写时就已经被决定了。函数创建时，`[[Scopes]]` 内部插槽一并被创建；函数执行时，会复制 `[[Scopes]]` 内部插槽，构建起执行环境的作用域链（外部环境记录）。



## 分类

- 全局作用域
- 函数作用域
- 块级作用域

`eval` 没有自己的作用域，在当前作用域内执行，因此可能会修改当前作用域的变量的值，造成安全问题。在严格模式，`eval` 函数内部声明的变量，不会影响到外部作用域。



## 作用域链

上下文中的代码在执行的时候，会以作用域链（scope chain）的方式查找一个变量。具体来说，首先会在当前执行上下文的作用域（环境记录）中查找，如果这个变量在当前作用域内没有被定义（这种变量被称为自由变量），那么就会根据外部环境记录在上级作用域中寻找这个变量。如果还没有找到，那就会在更上级的作用域中寻找，直到全局作用域。



## 优先级

函数作用域的优先级：本地变量>设置了默认值的传参>函数声明>未设置默认值的传参>变量提升
