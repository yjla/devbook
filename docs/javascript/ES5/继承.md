# 继承

- 原型链继承，父类的实例作为子类的原型，所有子类实例将共享父类的原型
  - 多个实例的引用类型属性指向相同，存在篡改的可能。
  - 无法传递参数

- 借用构造函数继承，通过改变 `this` 的指向，在字类的构造函数中直接调用将父类的属性和方法复制给子类

  - 可以继承多个父类，同时解决了不能向父类传参的缺点

  - 子类只能继承构造函数的属性和方法，无法继承父类的原型对象的属性和方法
  - 无法实现函数方法的复用（需要为每个子类实例对象都创建了一个函数方法）

- 组合继承：将原型链和构造函数组合起来使用的一种方式，借用构造函数实现父类方法的复用，使用原型链继承的方法继承父类的原型的属性和方法

  - 调用了两次父类的构造函数，造成了不必要的属性。

- 原型式继承：

  - 多个实例的引用类型属性指向相同，存在篡改的可能。
  - 无法传递参数

- 寄生式继承：

  - 多个实例的引用类型属性指向相同，存在篡改的可能。
  - 无法传递参数

- 寄生式组合继承：解决了组合继承的缺点，避免了创建不必要的属性。
