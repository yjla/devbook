# 内存管理

变量是没有数据类型的，值才有数据类型，变量可以随时持有任何类型的数据。



## 栈内存

栈内存是一块连续的存储空间，遵循着先进后出的原则。

- 基本数据类型的值保存在栈内存中，大小是固定的。
- 当把一个原始类型从原变量赋值到另一个变量时，实际上是在栈内存中开辟了一个新的空间，用于存放这个新变量。这两个变量之间相互独立，互不干扰，修改一个变量不会影响另一个。
- 栈内存中变量一般在它的当前执行环境结束就会被销毁被垃圾回收制回收。



## 堆内存

堆内存的堆指的不是数据结构中的堆，堆内存的结构更像是链表。

- 引用数据类型的值保存在堆内存中，大小不固定，并且在栈内存中保存了一个指向该引用类型的指针。
- 把引用类型从一个变量赋值给另一个变量时，实际上只是在栈内存中开辟了一个新的空间，用于存放这个原变量的指针。复制后，这两个变量指向同一个对象，因此，当通过其中一个变量修改了该引用类型之后，另一个变量也会发生变化。
- 堆内存中的变量只有在所有对它的引用都结束的时候才会被回收。
- 闭包中的变量保存在堆内存中



## 内存生命周期

1. 分配内存：可以通过初始化值和函数调用来分配
2. 读写内存
3. 释放内存（垃圾回收）



## 内存泄漏

常见的内存泄漏场景：

- 意外的全局变量
- 被遗忘的定时器和回调函数
- 不合理的闭包
- 无效的 DOM 引用



## V8 垃圾回收

- 新生代
- 老生代
- 全停顿
