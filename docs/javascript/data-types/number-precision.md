# 计算精度问题

为什么 `0.1+0.2 !== 0.3`？



## 原因

根据 IEEE 754，浮点数运算的时候需要先转成二进制，然后再进行运算。

具体过程：

- 整数部分采用 `/2` 取余法
- 小数部分采用 `*2` 取整法

而0.1转二进制会发生无限循环，引发舍入，所以就产生了误差。



## 解决办法

1. 使用 `Number.EPSILON` 最小精度值判断误差是否在该值范围内
2. 转为整数计算，计算后再转回小数
3. 转成字符串相加
