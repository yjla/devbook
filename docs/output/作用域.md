# 作用域

## 参数与同名 `var` 的提升

```js
function f(x) {
  console.log(x);
  var x = 1;
}
f(); // undefined
f(2); // 2
```

形参 `x` 和函数内 `var x` 是**同一个变量**。`var x = 1` 的声明被提升，但赋值留在原地。

- `f()`：未传参，`x` 是 `undefined`，打印时还没执行到 `x = 1`
- `f(2)`：形参 `x = 2`，`var x` 只是重复声明不会覆盖已有值，打印 `2`

:::info
`var x = 1` 等价于「提升 `var x`」+「执行到该行时 `x = 1`」。提升只提声明不提赋值，所以 `console.log(x)` 之前 `x` 仍是形参的值。
:::

## 函数声明的提升优先级最高

```js
function f(x) {
  console.log(x);
  var x = 1;
  function a() {
    return 3;
  }
}
f(2); // 2
```

打印 `2`。这里函数声明 `a` 与 `x` 无关，不影响 `x` 的值，所以是形参 `2`。

```js
function fn(x = 4) {
  var x = 5;
  console.log(x);
  var x = 1;
  function x() {
    return 3;
  }
}
fn(2); // 5
```

打印 `5`。执行顺序：形参 `x = 2` → 函数声明 `x` 提升 (此时 `x` 是函数) → 进入函数体执行 `var x = 5` 覆盖为 `5` → 打印 `5`。`var x = 1` 在打印之后才执行。

:::warning
带默认值参数 (`x = 4`) 与函数体内同名 `var` / `function` 声明混用，行为很反直觉。结论：打印的是执行到 `console.log` 那一刻 `x` 的最新赋值，本例是 `var x = 5`。
:::

## 参数默认值的暂时性死区

```js
function bar(x = y, y = 2) {
  return [x, y];
}
bar(); // 报错：Cannot access 'y' before initialization
```

参数按从左到右初始化。`x = y` 时 `y` 还未初始化，处于暂时性死区，抛 `ReferenceError`。

```js
let x = x; // ReferenceError: x is not defined
```

`let x = x` 中，等号右边的 `x` 引用的是正在声明、尚未初始化的同一个 `x`，触发暂时性死区。

:::info
**暂时性死区 (TDZ)**：`let` / `const` 声明的变量从作用域开始到声明语句之间不可访问。与 `var` 不同，TDZ 内访问会直接报错，而非返回 `undefined`。
:::

## 参数默认值形成独立作用域

```js
var x = 1;
function f(x, y = x) {
  console.log(y);
}
f(2); // 2
```

`y = x` 中的 `x` 指向**同名形参 `x`** (值为 `2`)，而非外层 `var x = 1`，所以打印 `2`。

```js
let x = 1;
function f(y = x) {
  let x = 2;
  console.log(y);
}
f(); // 1
```

打印 `1`。一旦设置了参数默认值，参数会形成一个**单独的作用域**。`y = x` 在这个参数作用域里求值，此时函数体的 `let x = 2` 还不可见，于是 `x` 向外层找到 `let x = 1`，所以 `y` 是 `1`。

:::info
一旦函数有参数默认值，参数列表会形成一个独立作用域 (context)，介于外层作用域和函数体作用域之间。默认值表达式在这个作用域里求值，看不到函数体内用 `let` / `const` 声明的同名变量。初始化结束后该作用域消失。
:::
