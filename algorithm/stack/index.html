堆栈

结论先行:栈和堆都是「故意限制访问方式,换取某个操作变快」的数据结构。

  • 栈 (Stack):只能从一端进、同一端出,后进先出 (LIFO)。天生适合处理「最近一次」「对称匹配」这类问题。
  • 堆 (Heap) / 优先队列:不管怎么塞,取的时候永远先拿到最大 (或最小) 的那个,取最值 O(1)、增删 O(log n)。天生适合「动态维护最值」「TopK」。
信息

名字辨析:这里的「堆」是数据结构的 二叉堆,和 JS 运行时内存里的「堆 (heap memory)」是两回事,别混。

一、栈​

生活类比​

一摞盘子:洗好的盘子一个个往上叠,用的时候只能从最上面拿。最后放上去的盘子最先被取走——这就是后进先出。你没法从中间抽一个,只能动栈顶。

什么时候该想到栈​

  • 需要处理 成对、对称 的结构:括号匹配、HTML 标签闭合。
  • 需要回看 「最近一个还没处理的东西」:表达式求值、撤销操作、函数调用栈。
  • 需要为每个元素找 「下一个 / 上一个更大 (更小) 的元素」 → 单调栈。
  • 一句话:凡是「后来的先算、算完再回头处理前面的」,就是栈。

例题一:有效括号 (20)​

题目:给一个只含 ()[]{} 的字符串,判断括号是否合法配对。比如 "([])" 合法,"([)]" 不合法。

合法的关键是 最近打开的括号,必须最先被关闭——这正是后进先出。所以用栈存「等待闭合的左括号」:遇左括号压栈,遇右括号就看 栈顶 那个左括号配不配得上。

信息

生活类比:套娃必须从里往外拆。一层层套起来的俄罗斯套娃,拆的时候只能从最里面 (最后套上的) 那个开始。右括号要找的搭档,一定是「当前还没合上的、最靠里的那个左括号」,也就是栈顶。

解题思路 1-2-3:

function isValid(s) {
  // 第一步:用一个映射记住「右括号 → 它该配的左括号」
  const pairs = {
    ')': '(',
    ']': '[',
    '}': '{',
  };

  // 第二步:准备一个栈,存还没闭合的左括号
  const stack = [];

  // 第三步:逐个字符扫描
  for (const ch of s) {
    if (ch === '(' || ch === '[' || ch === '{') {
      // 左括号:压栈,等待将来被闭合
      stack.push(ch);
    } else {
      // 右括号:看栈顶的左括号配不配
      const top = stack.pop();
      if (top !== pairs[ch]) {
        return false; // 配不上 (或栈本来就空,pop 出 undefined),不合法
      }
    }
  }

  // 第四步:扫完后栈必须空,否则还有左括号没被闭合
  return stack.length === 0;
}

走一遍 "([])":

当前字符动作栈
(压栈['(']
[压栈['(', '[']
]栈顶 [ 匹配 ],弹出['(']
)栈顶 ( 匹配 ),弹出[]

扫完栈空 → 合法。

注意

两个易漏的边界:① 右括号来时 栈是空的(如 ")"),pop() 出 undefined,和任何左括号都不等,返回 false,天然处理了。② 扫描结束 栈非空(如 "("),说明有左括号没闭合,所以最后要判 stack.length === 0,不能只在循环里判。

例题二:每日温度 (739) —— 单调栈​

题目:给每天的温度数组 T,对每天求「还要等几天才会更暖」。比如 [73,74,75,71,69,72,76,73] → [1,1,4,2,1,1,0,0],最后两天后面没有更高温,记 0。

暴力是每天往后找第一个更高温,O(n²)。单调栈能做到 O(n):栈里存 「还在等更高温的那些天的下标」,且对应温度从栈底到栈顶递减。新的一天温度更高时,就能一次性「结算」掉栈顶所有比它低的天。

信息

生活类比:排队等叫号的「踢馆」。一排人按个子从高到低站着 (栈),来一个新人:只要他比队尾的人高,队尾那个就「等到了对手」被请出去结算;一直比到队里没人比他矮,他再站到队尾等。每个人只进队一次、出队一次,所以总共才 O(n)。

解题思路 1-2-3:

function dailyTemperatures(T) {
  // 第一步:答案先全填 0 (后面没有更高温的天,就保持 0)
  const answer = new Array(T.length).fill(0);

  // 第二步:栈里存「还没等到更高温的天」的下标,
  //         保证栈中对应温度从栈底到栈顶递减
  const stack = [];

  // 第三步:遍历每一天 i
  for (let i = 0; i < T.length; i++) {
    // 第四步:当前温度比栈顶那天高,就能为栈顶那天结算答案
    while (stack.length > 0 && T[i] > T[stack[stack.length - 1]]) {
      const prevDay = stack.pop();
      answer[prevDay] = i - prevDay; // 等了 i - prevDay 天
    }

    // 第五步:当前这天还没等到更高温,压栈继续等
    stack.push(i);
  }

  return answer;
}

核心直觉:一个下标被弹出的那一刻,就是它「等到更高温」的那天,差值就是答案。每个下标只压一次、弹一次,所以是 O(n)。

提示

单调栈的触发信号:题目问「下一个 (上一个) 更大 / 更小的元素」。栈里存下标 (而非值),好处是既能拿到值 T[i],又能算距离 i - prevDay。

二、堆 / 优先队列​

生活类比​

急诊分诊台:病人不是按来的先后看病,而是按 病情严重程度 排。新病人随时来,但叫号时永远先叫最危急的那个。堆就是这样一个「随时能塞人、每次总能取出优先级最高者」的结构。

什么时候该想到堆​

  • 求 第 K 大 / 第 K 小、TopK 高频。
  • 需要 动态地、反复地 拿当前最值(数据还在不断进来)。
  • 合并多个有序序列(如合并 K 个有序链表),每次取各路最小。
  • 信号:发现自己「想要个东西,既能不断加,又能随时 O(1) 拿到最值」——排序做不到 (排完又变),这时候用堆。
信息

JavaScript 没有内置堆,刷题要么手写一个二叉堆,要么用数组模拟。下面给一个最小堆 (MinHeap) 的可读实现,核心就两个操作:插入后 上浮 (siftUp)、取出后把末尾补到顶再 下沉 (siftDown),始终维持「父节点 ≤ 子节点」。

例题:数组中第 K 个最大元素 (215)​

题目:无序数组里找第 k 大的元素。如 [3,2,1,5,6,4], k=2 → 第 2 大是 5。

排序后取 nums[n-k] 是 O(n log n)。更优的思路:维护一个大小为 k 的最小堆,堆里始终只留「目前最大的 k 个数」。堆顶是这 k 个里最小的,也就是 第 k 大。遍历完整个数组,堆顶即答案,复杂度 O(n log k)。

信息

生活类比:擂台只留前 k 名。擂台上最多站 k 个人,门口排队挑战。新人来了:不满 k 人就直接上;满了就和 当前最弱者 (堆顶) 比,赢了就把最弱者换下去,输了就回家。全程结束,台上最弱的那个 (堆顶) 正好是第 k 强。

解题思路 1-2-3:

// 第一步:实现一个最小堆,只需 push (插入) 和 pop (弹出最小)
class MinHeap {
  constructor() {
    this.data = []; // 用数组存完全二叉树:下标 i 的左右孩子是 2i+1、2i+2,父是 (i-1)/2
  }

  size() {
    return this.data.length;
  }

  peek() {
    return this.data[0]; // 堆顶,即最小值
  }

  push(val) {
    // 新元素先放末尾,再不断和父节点比、比父小就上浮
    this.data.push(val);
    let i = this.data.length - 1;
    while (i > 0) {
      const parent = Math.floor((i - 1) / 2);
      if (this.data[i] >= this.data[parent]) break; // 父已更小,位置对了
      [this.data[i], this.data[parent]] = [this.data[parent], this.data[i]];
      i = parent;
    }
  }

  pop() {
    // 取走堆顶,把末尾元素挪到顶,再不断和较小的孩子比、比孩子大就下沉
    const top = this.data[0];
    const last = this.data.pop();
    if (this.data.length > 0) {
      this.data[0] = last;
      let i = 0;
      const n = this.data.length;
      while (true) {
        const left = 2 * i + 1;
        const right = 2 * i + 2;
        let smallest = i;
        if (left < n && this.data[left] < this.data[smallest]) smallest = left;
        if (right < n && this.data[right] < this.data[smallest]) smallest = right;
        if (smallest === i) break; // 父已最小,位置对了
        [this.data[i], this.data[smallest]] = [this.data[smallest], this.data[i]];
        i = smallest;
      }
    }
    return top;
  }
}

function findKthLargest(nums, k) {
  // 第二步:用最小堆维护「目前最大的 k 个数」
  const heap = new MinHeap();

  // 第三步:遍历每个数
  for (const num of nums) {
    heap.push(num);
    // 第四步:堆超过 k 个,就弹掉最小的——留下的永远是最大的 k 个
    if (heap.size() > k) {
      heap.pop();
    }
  }

  // 第五步:堆顶是这 k 个里最小的,正是第 k 大
  return heap.peek();
}

走一遍 [3,2,1,5,6,4], k=2:

当前数入堆后超过 2 个?弹出最小后堆内容
3[3]否[3]
2[2,3]否[2,3]
1[1,3,2]是弹 1 → [2,3]
5[2,3,5]是弹 2 → [3,5]
6[3,5,6]是弹 3 → [5,6]
4[4,6,5]是弹 4 → [5,6]

堆顶 5 → 第 2 大是 5。

提示

为什么求「第 k 大」反而用「最小堆」?因为要 守住门槛:堆里留最大的 k 个,而最容易被淘汰的是其中最小的那个,把它放堆顶就能 O(1) 比较、O(log k) 换掉。求第 k 小则反过来用最大堆。记法:求大用小堆、求小用大堆。

小结​

  • 栈 = 后进先出,处理「最近一次」「对称匹配」。
    • 有效括号:左括号压栈,右括号配栈顶;扫完栈空才合法。
    • 每日温度 / 单调栈:栈存下标、温度递减,问「下一个更大元素」就想它,O(n)。
  • 堆 = 随时取最值,处理「动态最值」「TopK」,取最值 O(1)、增删 O(log n)。
    • 第 K 个最大:大小为 k 的最小堆守门槛,堆顶即第 k 大,O(n log k)。
    • 口诀:求第 k 大用最小堆,求第 k 小用最大堆。
  • JS 无内置堆,记住二叉堆用数组存,靠 上浮 / 下沉 维持堆序即可手写。
